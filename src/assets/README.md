# src/assets

앱에서 쓰는 정적 에셋. 대부분은 그대로 쓰는 이미지/사운드지만,
아래 항목은 생성·처리 이력과 코드 결합이 있어 따로 적어 둔다.

## bezel-frame.png — 큰 화면 모드 베젤 프레임

`body.large-screen .screen-wrap` 이 `border-image` 로 쓰는 픽셀아트 베젤
(모니터 이미지 대신 창을 채우는 레트로 프레임). 가운데는 완전 투명이라
그 자리로 어두운 CRT 화면(`--screen-dark`)과 뷰가 비쳐 보인다.

**현재 파일**: 450×410, 테두리 두께 위/아래 ≈ 16px, 좌/우 ≈ 20px, ~45KB.

### 생성 절차 (재현용)

1. 이미지 생성 AI(Gemini)로 "속이 빈 사각 베젤" 픽셀아트를 뽑는다.
   - 투명을 못 내므로 가운데는 마젠타(`#FF00FF`) 같은 단색으로 채워 받는다.
2. 배경 키아웃 → 투명화 (Pillow):
   - 무채색(회색 체크무늬·흰 반짝이)과 마젠타를 알파 0 으로.
   - "무채색 또는 마젠타면 삭제, 따뜻한 색이면 유지" 규칙으로 베젤만 남긴다.
3. 투명 영역으로 crop 후, **알파 미리곱(premultiply)** 상태에서 LANCZOS 로
   폭 ~450px 까지 축소한다(미리곱 안 하면 가장자리에 색 번짐/헤일로 발생).

### ⚠️ 코드 결합 — 재생성 시 반드시 확인

`styles.css` 의 `border-image-slice: 16 20` (위/아래 16, 좌/우 20) 은
**이 에셋의 테두리 두께(px)에 직접 묶여 있다.** 빌드 스텝이 없어 자동 검증이
안 되므로, 에셋을 다른 크기/두께로 다시 만들면 슬라이스가 조용히 어긋나
베젤이 깨진다. **재생성하면 새 테두리 두께를 다시 측정해 슬라이스를 갱신할 것.**

> 참고: 현재 에셋은 상하(16)와 좌우(20) 두께가 달라, 4면 동일 `border-width`
> 로 그리면 코너·픽셀 스텝이 상하/좌우에서 비대칭으로 보인다. 다시 만들 때
> **네 변 두께를 같게** 하면 슬라이스도 대칭값이 되어 이 문제가 사라진다.
